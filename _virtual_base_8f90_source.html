13  !> This is a generic data structure to virtualize pieces
14  !! of memory in 2 distinct ways:
15  !!
16  !! 1) Virtualize remote memory
17  !! This concerns memory residing on another process.
18  !! Typically, these pieces are subsets of certain model
19  !! and exchange data and lookup tables are kept with the
20  !! data to manage their mapping. The stage(s) at which
21  !! to synchronize the virtual memory is stored as well.
22  !!
23  !! 2) Virtualize local memory
24  !! In this case no virtual memory item is created, no
25  !! lookup tables and synchronization are necessary.
26  !! The virtual memory item will be pointed to the
27  !! original memory location at the requested
28  !! synchronization stage.
29  !<
30  type, abstract, public :: virtualdatatype
